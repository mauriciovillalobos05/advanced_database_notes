# Session – 2026-03-03.

## Topics covered
* SQL Window Functions core concepts
* The `OVER` clause and its modifiers (`PARTITION BY`, `ORDER BY`, `ROWS BETWEEN`)
* Window framing (defining specific row ranges to aggregate)
* Ranking functions (`ROW_NUMBER()`, `RANK()`, `DENSE_RANK()`)
* Filtering with window functions using subqueries

## What I understood
* Window functions perform calculations across a set of table rows related to the current row, without collapsing the result set like a standard `GROUP BY` does.
* The `OVER()` clause is the engine of a window function. It defines the "window" of data being looked at:
  * Leaving it empty (`OVER()`) applies the function to the whole table.
  * `PARTITION BY` divides the data into separate groups before applying the function.
  * `ORDER BY` creates a logical sequence within the window, which is essential for running totals or cumulative sums.
  * `ROWS BETWEEN` allows for fine-tuning the window frame relative to the current row (e.g., a rolling 3-row average).
* Ranking functions enumerate rows differently based on ties: `ROW_NUMBER()` is strictly sequential, `RANK()` skips numbers after a tie, and `DENSE_RANK()` never skips numbers.
* Because of the SQL order of execution, window functions cannot be used directly in a `WHERE` clause. To filter by a window function's result, the calculation must be placed inside a subquery (or CTE).
