# Session – 2026-02-10 PR

## Topics Covered
- 
These initial lessons focus on the `SELECT` statement and how to retrieve and filter data from a single database table.

### Lesson 1: SELECT Queries 101
* **The Goal:** Learn how to fetch data from a table.
* **Key Commands:** `SELECT` and `FROM`.
* **What you learn:** How to retrieve specific columns by name (e.g., `SELECT title, year FROM movies;`) and how to retrieve *every* column at once using the asterisk wildcard (e.g., `SELECT * FROM movies;`).

### Lesson 2: Queries with Constraints (Part 1)
* **The Goal:** Learn how to filter rows of data so you only get back what you need.
* **Key Command:** `WHERE`.
* **What you learn:** How to use basic mathematical operators to filter numerical and text data. This includes finding exact matches (`=`), exclusions (`!=`), and ranges (`<`, `>`, `<=`, `>=`).

### Lesson 3: Queries with Constraints (Part 2)
* **The Goal:** Learn how to build more complex filters for text and multiple conditions.
* **Key Commands:** `AND`, `OR`, `BETWEEN`, `IN`, `LIKE`.
* **What you learn:** How to chain conditions together (e.g., finding a movie that is a comedy `AND` came out after 2010). You also learn how to match text patterns using `LIKE` and the `%` wildcard (e.g., finding any movie title containing the word "Matrix").

### Lesson 4: Filtering and Sorting Query Results
* **The Goal:** Learn how to organize the data once you've retrieved it.
* **Key Commands:** `ORDER BY`, `LIMIT`, `OFFSET`.
* **What you learn:** * `ORDER BY` allows you to sort your results alphabetically or numerically in ascending (`ASC`) or descending (`DESC`) order. 
    * `LIMIT` caps the number of rows returned (great for "Top 5" lists).
    * `OFFSET` skips a specified number of rows before returning data (this is the foundation of website pagination).

### Lesson 5: Reviewing SELECT Queries
* **The Goal:** Consolidate everything you've learned.
* **Key Concept:** Query Execution Order.
* **What you learn:** A practice lesson that forces you to write a single, complex query using all previous commands. It reinforces the strict grammatical order SQL requires: `SELECT` → `FROM` → `WHERE` → `ORDER BY` → `LIMIT` / `OFFSET`.
