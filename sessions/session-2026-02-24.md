# Session – 2026-02-24.

## Topics covered
- **Aggregate Functions:** Introduction to functions that perform operations on multiple rows to return a single value (`COUNT`, `SUM`, `AVG`, `MIN`, `MAX`).
- **Grouping Data (`GROUP BY`):** How to divide rows into groups based on matching values in specific columns so that aggregates can be calculated per category.
- **Filtering Aggregates (`HAVING`):** Using the `HAVING` clause to filter data based on the results of aggregate functions, distinct from the `WHERE` clause.
- **Query Execution Order:** Understanding the sequence in which the database processes clauses (e.g., `FROM` -> `WHERE` -> `GROUP BY` -> `HAVING` -> `SELECT`).
- **Conditional Aggregation:** Using `CASE` statements inside aggregate functions to calculate metrics for specific subsets of data.
- **Advanced Grouping (`ROLLUP` & `CUBE`):** Generating automatic subtotals and grand totals across different dimensions of data.

## What I understood
- **The Role of Aggregates:** Aggregate functions allow us to summarize large datasets. For example, finding the total revenue (`SUM`), the number of employees in a department (`COUNT`), or the highest salary (`MAX`).
- **The "Golden Rule" of Grouping:** Whenever I use a `GROUP BY` clause, any column in my `SELECT` statement that is *not* wrapped in an aggregate function **must** be included in the `GROUP BY` clause. 
- **WHERE vs. HAVING:** `WHERE` is used to filter individual rows *before* they are grouped together. `HAVING` is used to filter the groups *after* the aggregations have been calculated. I cannot use aggregate functions inside a `WHERE` clause.
- **Order of Execution:** SQL doesn't read top-to-bottom. It first grabs the tables (`FROM`), filters the base data (`WHERE`), groups it (`GROUP BY`), filters the grouped data (`HAVING`), determines what to return (`SELECT`), and finally sorts it (`ORDER BY`).
- **Flexibility with CASE:** I can put `CASE WHEN` statements inside functions like `SUM()` or `AVG()` to conditionally count or average data without writing complicated subqueries.
- **ROLLUP and CUBE:** These are powerful tools for reporting. `ROLLUP` gives me hierarchical subtotals and a grand total, while `CUBE` calculates subtotals for every possible combination of my grouped columns.
