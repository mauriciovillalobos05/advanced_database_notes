# Session – 2026-02-17

## Topics covered
- **Relational Database Basics:** Tables, Columns, Primary Keys (IDs), and Foreign Keys.
- **SQL JOINs:** Connecting tables using INNER, LEFT, RIGHT, FULL, CROSS, and SELF joins.
- **Data Subtraction / Filtering:** Finding data present in one table but missing from another.
- **Database Design:** Normalization rules to minimize duplicated data and improve database management.

## What I understood
- Databases structure data into tables and vertical columns, relying on Primary Keys to uniquely identify records and Foreign Keys to establish relationships between tables.
- Different JOINs serve distinct matching purposes: `INNER` for overlaps, `LEFT`/`RIGHT` to keep a base table plus any matches, `FULL` for everything, and `CROSS` for a Cartesian product (all combinations).
- To pull records that exist in Table A but *not* in Table B, I can use the `NOT EXISTS` operator, a `LEFT JOIN` combined with an `IS NULL` filter.
- Normalization is a fundamental design process used to eliminate data redundancy.
